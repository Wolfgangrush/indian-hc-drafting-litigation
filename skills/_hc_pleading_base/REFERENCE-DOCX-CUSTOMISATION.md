# `reference.docx` — Pre-customised pandoc template

The shipped `reference.docx` in this folder is **pre-customised for Bombay HC formatting** (per `bench-config/exemplars/bombay-hc.md` Section 9). The Drafter agent uses it via pandoc's `--reference-doc` flag to render every `.docx` output with locked Word styles.

---

## What's locked in the shipped reference.docx

| Element | Locked value |
|---|---|
| Paper size | A4 (21.0 × 29.7 cm) |
| Margins | 4 cm left · 2.5 cm right / top / bottom |
| Body (Normal) | Times New Roman 14pt · 1.5 line spacing · justified · 0.5 cm first-line indent |
| Heading 1 | TNR 14pt · **bold** · centered (court header, case-number, cover-page anchors) |
| Heading 2 | TNR 14pt · **bold** · centered · letter-spacing widened for the `F A C T S` effect |
| Heading 3 | TNR 14pt · **bold** · left-aligned (ground sub-headers, application titles) |
| Title style | TNR 14pt · **bold** · centered |
| Tables | `tblLayout = fixed` (honours column widths set in Markdown pipe-tables) |
| Page numbers | Centred at bottom, TNR 11pt |

Drafter writes Markdown using `# Heading 1` / `## Heading 2` / `### Heading 3` per the `_hc_pleading_base/SKILL.md` heading-discipline contract. Pandoc maps the headings to the locked styles. The render is filing-grade out of the box.

---

## How to override for a different bench

If you practise at a bench with different formatting (e.g., Delhi HC Original Side Rules 2018 — A4 + **double** spacing + advocate-name-enrolment-phone-email at foot, or Punjab & Haryana HC — double-spaced watermarked paper), supply a per-case override:

1. Copy the shipped reference.docx into your case folder:
   ```bash
   cp <plugin>/skills/_hc_pleading_base/reference.docx <your-case-folder>/reference.docx
   ```
2. Open in Word / LibreOffice and modify the Normal style (line spacing, font) and section margins per your bench's Practice Notes.
3. Save. The Drafter checks for `<case-folder>/reference.docx` first and uses it if present; otherwise falls back to the shipped one.

## Regenerating the shipped reference.docx (plugin maintainers only)

The shipped reference.docx is generated by the script at `${CLAUDE_PLUGIN_ROOT}/skills/_hc_pleading_base/build_reference_docx.py` (committed alongside this file). Run:

```bash
cd <plugin-folder>/skills/_hc_pleading_base
python3 build_reference_docx.py reference.docx
```

The script uses `python-docx` (`pip install python-docx`) to lock the Word styles. Do not hand-edit the shipped reference.docx — re-run the script if the styles need updating, so the change is reproducible.

---

## What NOT to put in reference.docx

- No header / footer text content. The reference is a STYLE template only.
- No case-specific content. The shipped reference contains a short demonstration body to show what the styles look like; pandoc replaces ALL content when rendering the draft.
- Do not move the file or change the filename.

---

## How the Drafter uses it

```bash
pandoc draft-v1.md -o draft-v1.docx \
  --reference-doc="${CLAUDE_PLUGIN_ROOT}/skills/_hc_pleading_base/reference.docx" \
  --from=markdown+pipe_tables+raw_tex
```

Pandoc reads the styles from the reference.docx and applies them to every section of the draft.

---

## Pandoc is required (not optional)

```bash
brew install pandoc       # macOS
sudo apt install pandoc   # Debian / Ubuntu
```

The Drafter does not fall back to plain `python-docx` rendering — that produces v0.1.0-grade render defects (title not bold, FACTS left-aligned, Index table column-headers wrap vertically). Pandoc with the shipped reference.docx is the documented render path.
